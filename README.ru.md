# Telegram WEBM Sticker Converter

[English](README.md) | [Русский](README.ru.md)

Скачать последний релиз: [последний релиз](https://github.com/miffril/tg-stickers-converter/releases/latest)

Простая консольная утилита на C# для создания Telegram WEBM-стикеров и эмодзи путем конвертации GIF, MP4, AVIF или PNG-анимаций в оптимизированные WebM-видео с помощью FFmpeg.

Чтобы добавлять WEBM-стикеры (видео-стикеры) в Telegram, нужно создать набор видео-стикеров.

## О проекте

Этот проект появился, когда я столкнулся с проблемой конвертации прозрачных GIF в WEBM с изменением размера: по контуру изображения появлялись артефакты из-за неравномерного апскейла или даунскейла альфа-канала. Со временем я добавил и другие функции.

Зачем было делать еще один велосипед? Другие решения не закрывали все проблемы. У одного сервиса была одна проблема, у другого — другая, а некоторые проблемы встречались вообще у всех сразу. Во всех сервисах, которые я видел и где используется FFmpeg, была одна и та же проблема: артефакты при конвертации GIF с прозрачностью в WEBM. Почти везде приходится вручную подгонять размер и вес файла под Telegram, потому что сами сервисы были сделаны не для создания стикеров. У стикер-ботов встречаются те же проблемы, с которыми столкнулся и я, например артефактная полоса на iOS у неквадратных стикеров.

## Состояние проекта

Проект поддерживается вяло, потому что основные нужные мне функции уже реализованы. Пока я не планирую добавлять новые фичи, так как у меня не так много идей. Если вам что-то нужно, открывайте discussion. Если будет достаточный интерес, возможно, в будущем я сделаю GUI.

Большинство основных багов уже исправлено, но я не тестировал каждый edge case, поэтому почти наверняка остаются еще невыявленные проблемы.

## Известные проблемы

- FPS у GIF рассчитывается неточно, если задержки между кадрами различаются, из-за чего анимация после конвертации может выглядеть дерганой.
- FPS у эмодзи может вести себя странно. Я не знаю точную причину; это больше похоже на проблему клиентского рендеринга.
- Поддержка AVIF пока не была нормально протестирована.

## Возможности

- Извлечение и масштабирование кадров из GIF, MP4, AVIF или PNG-последовательности
- Для MP4 автоматически определяется и сохраняется исходный FPS
- Для AVIF автоматически определяются анимированные потоки и FPS из метаданных
- Сохраняются исходные пропорции изображения без растягивания
- Можно задать размер выходного кадра в пикселях (по умолчанию: 512 px по большей стороне)
- Опциональное добавление прозрачного паддинга до квадратного холста
- Быстрый режим эмодзи (`-e`) для 100x100
- Опциональная плавная обводка с настраиваемыми цветом и толщиной
- **Поддерживается опциональное размытие обводки с настраиваемым радиусом**
- **Полная поддержка AVIF**: обнаружение нескольких потоков, объединение цветового и альфа-каналов для прозрачности
- Корректный расчет FPS по таймингам GIF, метаданным AVIF или использование исходного FPS для MP4
- Кодирование кадров в VP9 WebM с поддержкой альфа-канала (только для GIF/PNG/AVIF)
- Подбор CRF до тех пор, пока размер файла не станет меньше 256 КБ (или 64 КБ в режиме эмодзи)
- Строгий режим по умолчанию: GIF/MP4/AVIF длиннее 3 секунд отклоняются
- Опциональный режим `--allow-speedup`: принимает GIF/MP4/AVIF длиной 3-5 секунд и ускоряет их до лимита Telegram в 3 секунды
- Проверка итоговой длительности и итеративное увеличение ускорения, пока результат не станет `<= 3.0` секунды
- Вывод итогового достигнутого множителя ускорения в режиме ускорения

## Требования

- .NET Framework 4.7.2 (требуется поддержка WPF)
- `ffmpeg.exe` и `ffprobe.exe` должны находиться в той же директории, что и исполняемый файл
- **FFmpeg должен быть собран с поддержкой AVIF (libaom/libdav1d) для работы с AVIF**

## Использование

```
Converter.exe [options]
```

### Опции

| Опция                | Описание |
|----------------------|----------|
| `-i`, `--input`      | Входной GIF, MP4, AVIF или PNG-файл (для PNG-последовательности) |
| `-o`, `--output`     | Выходной WebM-файл |
| `-c`, `--crf-step`   | Шаг увеличения CRF (по умолчанию: `2`) |
| `-b`, `--border`     | Добавить обводку к кадрам (по умолчанию: выключено) |
| `--border-size`      | Толщина обводки в пикселях (по умолчанию: `2`) |
| `--border-color`     | Цвет обводки в hex-формате (по умолчанию: `#FFFFFF`) |
| `--blur <value>`     | Радиус размытия обводки (целое число; по умолчанию размытие отключено) |
| `--fps`, `--input-fps` | Входной FPS для извлечения кадров (по умолчанию: `10`). Автоматически вычисляется для GIF/MP4/AVIF |
| `--target-fps`, `--output-fps` | Целевой FPS выходного WebM (по умолчанию: `30`, стандарт Telegram) |
| `-s`, `--size`       | Целевой размер в пикселях по большей стороне (по умолчанию: `512`) |
| `-p`, `--pad`        | Добавить паддинг до квадратного холста (по умолчанию: выключено) |
| `-e`, `--emoji`      | Установить размер 100x100 для эмодзи |
| `--allow-speedup`    | Разрешить GIF/MP4/AVIF длиной 3-5 секунд и ускорить их, чтобы уложиться в 3 секунды |
| `-h`, `--help`       | Показать справку |

## Примеры

Конвертация GIF в WebM с настройками по умолчанию (с сохранением пропорций, максимум 512 px):
```
Converter.exe -i animation.gif
```

Конвертация GIF в WebM с пользовательским именем выходного файла:
```
Converter.exe -i animation.gif -o result.webm
```

Конвертация анимированного AVIF в WebM с прозрачностью (output: animation.webm):
```
Converter.exe -i animation.avif
```

Конвертация AVIF в emoji WebM (output: sticker.webm):
```
Converter.exe -i sticker.avif -e
```

Конвертация GIF в WebM с паддингом до квадратного холста:
```
Converter.exe -i animation.gif --pad
```

Конвертация MP4 в WebM (с сохранением исходного FPS, output: video.webm):
```
Converter.exe -i video.mp4
```

Конвертация MP4 длиной 3-5 секунд с ускорением под лимит Telegram:
```
Converter.exe -i video.mp4 --allow-speedup
```

Конвертация GIF длиной 3-5 секунд с ускорением и выводом итогового ускорения:
```
Converter.exe -i animation.gif --allow-speedup
```

Конвертация GIF в emoji 100x100 (output: animation.webm):
```
Converter.exe -i animation.gif -e
```

Конвертация с пользовательским размером и белой обводкой:
```
Converter.exe -i animation.gif -s 256 -b
```

Конвертация с пользовательскими цветом и толщиной обводки:
```
Converter.exe -i animation.gif -b --border-size 4 --border-color "#FF0000"
```

Конвертация с пользовательским размытием обводки:
```
Converter.exe -i animation.gif -b --blur 2
```

Конвертация PNG-последовательности с пользовательским FPS:
```
Converter.exe -i "first_frame.png" --input-fps 24 --target-fps 30
```

Конвертация видео 60 FPS в совместимые с Telegram 30 FPS:
```
Converter.exe -i video60fps.mp4
```

Конвертация с пользовательским выходным FPS (не рекомендуется для Telegram):
```
Converter.exe -i animation.gif --target-fps 60
```

Показать справку:
```
Converter.exe --help
```

## Работа с FPS

Конвертер разделяет **входной FPS** (для извлечения кадров) и **целевой FPS** (для выходного файла):

### Входной FPS (`--fps` / `--input-fps`)
Определяет, как кадры извлекаются из исходного файла:
- **GIF**: автоматически вычисляется по задержкам кадров в метаданных
- **MP4**: автоматически определяется через ffprobe
- **AVIF**: автоматически определяется из метаданных потока
- **PNG-последовательность**: по умолчанию 10 FPS или пользовательское значение
- Можно переопределить параметром `--input-fps`

### Целевой FPS (`--target-fps` / `--output-fps`)
Определяет частоту кадров выходного WebM:
- **По умолчанию: 30 FPS** (стандарт Telegram)
- **Критично для iOS**: Telegram на iOS строго требует 30 FPS
  - Видео 60 FPS воспроизводятся в 2 раза медленнее на iOS
  - Другие значения FPS могут вызывать проблемы с воспроизведением
- FFmpeg автоматически корректирует кадры:
  - **Более высокий входной FPS** (например, 60 → 30): удаляет кадры
  - **Более низкий входной FPS** (например, 15 → 30): дублирует кадры
  - **Длительность остается неизменной**

## Работа с длительностью

- **Поведение по умолчанию**: GIF/MP4/AVIF длиннее 3 секунд отклоняются
- **Опциональное ускорение**: `--allow-speedup` принимает файлы длительностью **от 3 до 5 секунд** и ускоряет их, чтобы уложиться в строгий лимит Telegram в 3 секунды
- Конвертер **не** обрезает итоговый файл жестко через `-t 3`, потому что это может ломать бесшовные loop-анимации
- В режиме ускорения конвертер:
  1. Извлекает кадры за всю допустимую длительность исходника
  2. Кодирует WebM с более высокой входной частотой кадров
  3. Измеряет длительность готового файла через `ffprobe`
  4. Перекодирует с большим ускорением, если результат все еще длиннее 3.0 секунд
  5. Выводит итоговый множитель ускорения

Пример вывода:
```
Final speedup: 1.667x (5.000s -> 3.000s)
```

### Пример преобразования FPS
```bash
# Исходник 60 FPS → выход 30 FPS (стандартный случай)
Converter.exe -i video60fps.mp4
# Output: Input FPS: 60, Target output FPS: 30
# FFmpeg удалит каждый второй кадр

# Пользовательское извлечение кадров со стандартным выходом
Converter.exe -i video.mp4 --input-fps 15 --target-fps 30
# Извлекает 15 FPS, выдает 30 FPS (кадры дублируются)
```

## Подробности поддержки AVIF

Конвертер полностью поддерживает анимированные AVIF-файлы:

- **Обнаружение нескольких потоков**: автоматически определяет, какой поток содержит анимацию (в AVIF может быть несколько видеопотоков)
- **Поддержка альфа-канала**: объединяет отдельные цветовой и альфа-потоки через фильтр `alphamerge`
- **Определение частоты кадров**: автоматически определяет FPS из метаданных AVIF
- **Ограничение по длительности**: соблюдает максимальную длительность 3 секунды
- **Статический AVIF**: одиночные AVIF-изображения тоже поддерживаются

## Примечания

- **Входные GIF, MP4 и AVIF длиннее 3 секунд по умолчанию отклоняются. Используйте `--allow-speedup` для файлов длиной 3-5 секунд.**
- **Выходные видео по умолчанию кодируются в 30 FPS** (стандарт Telegram и требование iOS).
- Для MP4 используется автоматически определенный исходный FPS для извлечения кадров.
- Для AVIF FPS автоматически определяется по метаданным потока.
- Используйте `--target-fps`, чтобы переопределить выходной FPS (не рекомендуется для Telegram stickers/emoji).
- **Если имя выходного файла не задано, оно формируется из имени входного файла с расширением `.webm`** (например, `animation.gif` → `animation.webm`).
- Промежуточные кадры сохраняются как PNG в директорию `frames/` (с автоматической очисткой).
- Параметр `-s`/`--size` задает размер по большей стороне с сохранением исходных пропорций.
- По умолчанию изображение масштабируется под целевой размер без паддинга. Используйте `--pad`, чтобы центрировать изображение на квадратном прозрачном холсте.
- Параметр `--blur <value>` включает размытие обводки с указанным радиусом. Если не задан, обводка остается резкой.
- **Прозрачность поддерживается для GIF, PNG и AVIF. MP4 не поддерживает прозрачность.**
- **Если включен режим эмодзи (`-e`), лимит размера выходного файла снижается до 64 КБ. Иначе лимит составляет 256 КБ.**
- Конвертер не блокирует завершение; после выполнения он не ждет нажатия клавиши.
